# Coding standards

# Consistency: 
Use the same formatting, naming conventions, and structure throughout the project to make it look like a single developer wrote it.

# Readability: 
Write code that is easy to read and understand, even for someone who did not write it.

# Naming Conventions: 
Use descriptive and meaningful names for variables, functions, and classes to indicate their purpose.

# Structure and Formatting:
- Indentation: Use consistent indentation (spaces are generally preferred over tabs) to show the structure of the code.
- Whitespace: Use blank lines to separate logical sections and improve visual clarity.
- Line Length: Keep lines short to enhance readability.
- Braces: Use a consistent style for braces (e.g., placing them on a new line or the same line).
- Function/Method Size: Keep functions and methods small and focused on a single task, which makes them easier to understand, test, and maintain.
- Comments: Use comments to explain the "why" behind code, rather than the "what," and keep them updated.
